# EEG State Classifier

A project-based machine-learning pipeline for classifying simple EEG brain-wave states from frequency-domain features. The project started as a hardware + ML prototype using a portable EEG sensor and Arduino-style serial collection, then was refactored into a clean, reproducible Python package suitable for coursework and portfolio review.

> **Status:** coursework / research prototype. The included sample data is synthetic and is provided only so reviewers can run the repository end-to-end. Do not treat the model as a medical or safety-critical system.

## What this project demonstrates

- EEG signal preprocessing with a 0.5-40 Hz band-pass filter.
- Sliding-window feature extraction from raw one-channel EEG signals.
- Bandpower features for delta, theta, alpha, and beta frequency ranges.
- Ratio features such as alpha/beta, theta/beta, and slow/fast power.
- A reproducible scikit-learn pipeline using `StandardScaler` + `SVC`.
- Train/test evaluation with accuracy, classification report, confusion matrix, and saved model artifacts.
- A command-line workflow that can be run without a notebook.

## Quick start

```bash
git clone https://github.com/gitLamHoang/Classfying-brain-wave-state.git
cd Classfying-brain-wave-state
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
pytest -q
```

Generate demo data:

```bash
python scripts/generate_sample_data.py --output-dir data/sample
```

Train the baseline model:

```bash
python scripts/train_model.py \
  --features-csv data/sample/eeg_features_sample.csv \
  --model-out models/svm_eeg_state.joblib \
  --reports-dir reports
```

Run prediction on a raw recording:

```bash
python scripts/predict_window.py \
  --raw-txt data/sample/eeg_raw_awake_sample.txt \
  --model models/svm_eeg_state.joblib \
  --output-csv reports/sample_predictions.csv
```

## Using your original two-file training format

If your current training files are named like the original prototype:

- `testmo.csv`: awake / alert feature rows
- `testnham.csv`: sleepy / drowsy feature rows

train with:

```bash
python scripts/train_model.py \
  --awake-csv testmo.csv \
  --sleepy-csv testnham.csv \
  --model-out models/svm_eeg_state.joblib \
  --reports-dir reports
```

## Feature engineering

For each EEG window, the pipeline computes:

| Feature group | Description |
|---|---|
| Bandpower | `delta_power`, `theta_power`, `alpha_power`, `beta_power` |
| Ratios | `alpha_beta_ratio`, `theta_beta_ratio`, `delta_beta_ratio`, `theta_alpha_ratio`, `delta_alpha_ratio`, `slow_fast_ratio` |

The original code extracted similar frequency-band features, but the refactored version makes the feature order explicit and uses safe ratio calculations to avoid division-by-zero errors.

## Model

The baseline model is a scikit-learn pipeline:

```text
StandardScaler -> Support Vector Classifier
```

Why this design:

- Scaling keeps feature magnitudes comparable before SVM training.
- SVM is a strong baseline for small tabular datasets.
- The pipeline saves preprocessing and model weights together, so prediction uses the exact same transformations as training.

## Current limitations

- The public sample dataset is synthetic; it only proves the code pipeline runs.
- Real performance should be reported only on participant-level held-out sessions.
- The model currently uses one-channel features, not multi-channel spatial information.
- Labels such as `awake` and `sleepy` depend on the collection protocol and should be documented carefully.

## Roadmap
